Query creation and execution

The current focus of the project is to design a query creation interface that is intuitive and flexible for whatever sets of data someone might need. So far, the "create a query" page is functional, but it lacks form validation (ie it's possible to create a query with some of the variables set to "null", which makes the result unusable) and the ability to more directly edit the order of parameters (deleting parameters without needing to reset the entire form, changing the order of different condition sets, etc). The goal of this stage is to make the query system accessible to people without any experience with either Javascript or SQL, while still allowing for a direct SQL interface to be integrated for people who know SQL and would prefer to work with that syntax.
